아마도 C 언어가 그 후보 중에 빠질 수는 없을 것입니다.

C 언어는 그만큼 영향력이 큰 언어지만, 아주 간단한 언어이기도 합니다.
즉 배워야할 문법이 그렇게 많지가 않다는 뜻입니다.
이것은 빨리 배우기에 아주 좋은 장점입니다.

그리고 C 언어는 고수준 언어이기는 하지만, 아주 고수준은 아니기도 합니다.
그래서 컴퓨터의 내부 동작에 대한 이해가 필요하기도 합니다.

오늘날 프로그래밍은 파이썬과 같은 고수준 언어로 시작하는 것이 적지 않아보이지만, C 언어는 충분히 배울만한 가치가 있는 언어입니다.
왜냐면 프로그래밍을 하는 방법과 동시에, 컴퓨터의 내부 동작 또한 배울 수 있기 때문입니다.
그리고 이것을 배우고 나면, 다른 고수준 언어에서 감추고 있는 부분을 볼 수 있는 시야가 생길 것입니다.

C 프로그래밍 언어 책의 시작에는, 프로그래밍 언어는 프로그램을 만들어보며 배울 수 있다고도 합니다.
여기서도 이 구절을 따라, 작은 프로그램을 만들어보며 C 언어의 문법을 단계적으로 익혀보겠습니다.
이와 더불어 컴퓨터의 메모리 내부 동작 또한 같이 이해하게 될 것입니다.



# 프로그램 만들어보기

먼저, 앞으로 만들 할 일 관리 프로그램은 셸에서 다음과 같이 동작할 것입니다.

```shellsession
$ todo add foo bar baz
Added task 'foo'.
Added task 'bar'.
Added task 'baz'.
$ todo list
1: foo
2: bar
3: baz
```

즉 할 일을 여러 개 추가하는 기능과, 목록을 보는 기능을 만들 것입니다.

이 과정에서 C 언어의 기본적인 문법을 모두 익히게 될 것입니다.
그러면 본격적인 프로그래밍을 시작해봅시다.

## 프로그램 컴파일 하기

먼저 알아야 할 내용은 프로그램을 만드는 방법입니다.
우선 가장 간단한 프로그램으로서, 아무것도 하지 않는 빈 프로그램을 만들어보겠습니다.
다음을 `todo.c` 파일로 만듭니다.

```c
int main(void) {
  return 0;
}
```

이것은 `main`이라는 함수를 만든 것입니다.
C 프로그램은 항상 이 함수부터 시작합니다.
여기서 `main`과 함수란 나중에 자세히 살펴볼 것입니다.

이 함수가 하는 일은 숫자 0을 돌려주는 것입니다.
수학적으로 말하자면, 언제든 $0$을 가지는 함수로서 $f(x) = 0$ 같은 것입니다.
(물론 여기서의 함수는 수학적인 함수와는 차이가 있을 것입니다. 즉 비유적인 얘기입니다.)

앞서 언급한대로, C 컴파일러가 준비되어 있다고 해보겠습니다.
그러면 다음과 같이 컴파일<L>compile</L> 할 수 있습니다.

```shellsession
$ cc todo.c
```

그러면 기본적으로 `a.out` 파일이 만들어집니다.
이것을 실행해봅시다.

```shellsession
$ ./a.out
```

어떤 일이 일어날까요?
정말로 아무 일도 일어나지 않는지 확인해보세요.

앞서 돌려준다고 언급했던 `0`은 다음과 같이 종료 코드<L>exit code</L>로서 확인할 수 있습니다.
셸에서는 `$?` 변수에 방금 돌려준 종료 코드가 들어있습니다.

```shellsession
$ echo $?
0
```

이것이 정말로 `main` 함수에서 돌려준 값일까요?
한번 `main` 함수가 다른 값을 돌려주도록 수정해보세요.

```c
int main(void) {
  // [!code --]
  return 0;
  // [!code ++]
  return 1;
}
```

그리고 다시 셸에서 `$?` 변수로 종료 코드를 확인해보세요.
팁으로서 덧붙이자면, 컴파일과 실행은 다음과 같이 한 줄로 한번에 실행할 수 있습니다.

```shellsession
$ cc todo.c && ./a.out
$ echo $?
1
```

또 하나의 팁으로서, 셸에서는 명령어를 기억하고 있으므로, Ctrl+P 혹은 위 애로우를 눌러서 방금 실행한 명령어를 가져올 수 있습니다.
앞으로 컴파일한다는 뜻은 이런 과정을 거쳐 프로그램을 만든다는 뜻입니다.

## 컴파일러의 역할

이 컴파일러가 왜 필요한 것일까요?

컴퓨터는 사실 앞서 작성한 코드 그 자체를 이해할 수 없고, 다만 프로세서라고 부르는 장치가 이해할 수 있는 명령이 정해져있습니다.
실제로 우리가 컴파일한 결과인 `a.out`은 다음과 같은 명령이 있다는 것을 다음과 같이 볼 수 있습니다.

```shellsession
$ cc -g todo.c
$ objdump --disassemble=main --source --section=.text a.out
```

그러면 다음과 비슷한 결과가 나타날 것입니다.

```shellsession
int main(void) {
  return 1;
// [!code word:52800020]
 7a8:   52800020        mov     w0, #0x1                        // #1
// [!code word:d65f03c0]
 7ac:   d65f03c0        ret
```

이것은 `main` 함수를 기계어로 바꾼 결과입니다.
구체적으로는, `52800020`나 `d65f03c0` 같은 것이 프로세서가 이해할 수 있는 명령입니다.
여기서는 이것이 무슨 의미인지 다루지 않겠지만, 이 결과는 컴퓨터마다 다를 수 있습니다.
프로세서라고 부르는 장치는 제조사마다 다를 수 있으므로, 프로세서가 이해할 수 있는 명령 또한 다를 수 있기 때문입니다.
여기서 실행한 결과는 ARM 사의 프로세서를 이용한 맥에서 실행한 결과입니다.

즉 컴파일러란 일종의 번역기처럼, C 언어로 작성된 코드를 그러한 명령으로 옮겨주는 것입니다.
여기서 그 출발이 되는 C 코드를 소스 코드<L>source code</L>라고 부릅니다.

정리하자면, 프로그래밍이라면 프로세서가 직접 이해할 수 있는 명령을 내려서 원하는 일을 하는 것입니다.
물론 프로세서의 매뉴얼을 보며 그렇게 할 수도 있겠지만, C 컴파일러는 다음과 같은 두 가지 문제를 해결해줍니다.

1. 조금 더 인간에 가까운 언어로 프로그램을 작성할 수 있게 도와줍니다.
   이런 언어를 고수준<L>high-level</L> 언어라고 부릅니다.
   반대로 기계에 가까울수록 저수준<L>low-level</L>이라고 부릅니다.
1. 구체적인 프로세서 하드웨어를 신경 쓸 필요 없이, 다양한 프로세서 상에서 돌아가는 프로그램을 한번에 작성할 수 있게 도와줍니다.
   각 프로세서를 지원하는 C 컴파일러가 미리 만들어져있는 덕분입니다.

## 다른 코드 불러오기: include

C 언어에서는 다음과 같이 다른 코드를 불러와서 쓸 수 있습니다.
여기서는 가짜로 할 일이 있다고 치고 목록을 보여주는 부분부터 만들어보겠습니다.

```c
// [!code ++:2]
#include <stdio.h>

int main(void) {
  // [!code ++:1]
  printf("1: Learn C\tTo-do\n");

  return 0;
}
```

컴파일하고 실행하면 다음과 같을 것입니다.

```shellsession
1: Learn C
```

여기서는 `stdio.h` 헤더 파일을 불러오는데, 여기에 `printf` 함수가 있기 때문입니다.
여기서 헤더 파일이 무엇인지는 나중에 살펴볼 것입니다. (심지어 만들기도 해볼 것입니다.)

이 `printf`는 `"1: Learn C\n"`를 받아서 셸에 보여줍니다.
이 따옴표로 감싼 것을 문자의 나열이라는 뜻으로 문자열, 혹은 스트링<L>string</L>이라고 부릅니다.

그리고 `\t`와 `\n`은 각각 탭 문자와 줄바꿈 문자로서, 기존의 의미에서 '탈출'한 문자이기 때문에 이스케이프<L>escape</L> 문자라고도 부릅니다.
줄바꿈은 다음과 같이 쓸 수는 없다는 점을 참고하세요.

```c
  // [!code error:2]
  printf("1: Learn C\tTo-do
  ");
```

또는 다음과 같이 쓸 수도 있습니다.

```c
  // [!code --:1]
  printf("1: Learn C\tTo-do\n");
  // [!code ++:1]
  printf("%d: %s\t%s\n", 1, "Learn C", "To-do");
```

여기서 `%d`와 같은 것은 일종의 빈 자리로서, 뒤에 따라오는 값이 그 자리를 채웁니다.
구체적으로는, `%d`에는 숫자가, `%s`에는 스트링이 온다는 뜻입니다.
이러한 `%d`는 포맷<L>format</L>을 지정한다고 하며, 그래서 함수의 이름이 그 앞글자인 f를 따서 `printf`입니다.
우리는 이 방식을 고수할 것입니다.

## 이름 짓기 1: 상수

프로그래밍에는 이름이 없는 값을 최대한 지양하라는 원칙이 있습니다. (TODO: 출처?)
이 원칙은 앞으로도 따를 원칙입니다.

이것은 다음과 같은 방식으로 따를 수 있습니다.
종료 코드 `0`은 셸에서 성공적인 종료를 나타내는데, 이를 따라 다음과 같이 쓸 수도 있습니다.

```c
#include <stdio.h>
// [!code ++:1]
#include <stdlib.h>

int main(void) {
  printf("%d: %s\t%s\n", 1, "Learn C", "To-do");

  // [!code --:1]
  return 0;
  // [!code ++:1]
  return EXIT_SUCCESS;
}
```

이 `EXIT_SUCCESS`는 `stdlib.h` 파일에서 제공하는데, 값이 고정되어 있으므로 상수<L>constant</L>라고도 부릅니다.
이렇게 함으로써, 코드를 읽을 때 `0`의 의미가 무엇인지 고민하는 대신, 그 의미를 바로 파악할 수 있게 됩니다.

## 이름 짓기 2: 변수

이번엔 우리가 직접 이름을 지어봅시다.
이처럼 다음과 같이 변수<L>variable</L>를 만들어 이름을 지어줄 수 있습니다.

```c
int main(void) {
  // [!code ++:3]
  int id = 1;
  char name[] = "Learn C";
  char status[] = "To-do";

  // [!code --:1]
  printf("%d: %s\t%s\n", 1, "Learn C", "To-do");
  // [!code ++:1]
  printf("%d: %s\t%s\n", id, name, status);
}
```

여기서 `id` 변수에 주목해봅시다.
`=` 기호는 일종의 연산자로서, 오른쪽 값을 넣습니다.
이를 초기화<L>initialize</L>이라고 부릅니다.
즉 이제 `id` 라고 하면, `1`이라는 값을 나타내는 것입니다.

C 언어는 다음과 같이 분리할 수도 있습니다.

```c
  int id;
  id = 1;
```

이는 `id` 변수를 선언<L>declare</L>한다고 부르고, `1`을 할당<L>assign</L>한다고 부릅니다.

하지만 C 언어에서는 되도록이면 선언만 하지 말고 바로 초기화하는 것이 좋습니다. (TODO: callout(?))
왜냐면 선언만 한 변수에는 알 수 없는 값이 들어있기 때문입니다.
예전에 다른 프로그램이 썼다가 지워지지 않은 값일 수도 있습니다.
이런 값은 쓰레기 값<L>garbage</L>라고 부르고, 절대 이런 값을 이용하는 프로그램을 만들어서는 안 됩니다. (TODO: 쓰레기값 용어 출처?)

나머지도 이와 같습니다.
앞에 붙은 `int`와 같은 것들은 곧 살펴볼 것입니다.

이제 각각의 값이 어떤 의미를 갖는지 이름으로 파악할 수 있습니다.
예를 들어, 숫자 `1`은 할 일의 순번이고, `"Learn C"`는 그 이름이고 이런 식입니다.

## 메모리 크기 정하기: 타입

변수란 기본적으로 컴퓨터에서 메모리 공간을 빌려온 것입니다.
`int`와 같은 타입<L>type</L>으로 얼마나 많은 공간을 빌려올지 정할 수 있습니다.

(TODO: 그림)

정확한 크기는 플랫폼마다 다릅니다.
즉 구체적인 컴퓨터 하드웨어에 따라 다릅니다.
따라서 정확히 32비트의 공간을 빌려오겠다는 이름 대신, `int`, `short`, `long` 등의 이름을 갖고 있습니다.
`short`는 좀더 작은 숫자를, `long`은 좀더 큰 숫자를 나타내고, 이에 따라 더 많은 메모리 공간을 빌립니다.
`char`는 문자를 나타내는데, 이 타입은 여기서 예외로서 정확히 1 바이트를 차지하는 걸로 약속됐습니다.

한편, 스트링은 문자의 나열인데 개수가 항상 똑같지 않으므로 크기가 정해진 것이 아닙니다.
그래서 스트링은 문자 타입을 여러 개 나열한 배열을 사용합니다.
배열은 연속된 메모리 공간을 빌린 것을 말합니다.
스트링을 위해 쓰인 `char []` 타입은 연속된 메모리 공간을 빌려오겠다는 뜻입니다.
이것은 배열<L>array</L> 타입이라고도 부릅니다.
즉 다음과 같은 코드는 스트링을 담기 위한 사이즈를 계산해서 빌려옵니다.

```c
  char name[] = "Learn C";
```

## 코드 분리하기: 함수

(TODO: 함수란: 일을 추상화하는것, 함수의 목적 설명 추가)

앞서 `main` 함수를 만들었듯이, 다른 함수도 만들어볼 수 있습니다.
여기서는 할 일을 프린트하는 작업을 별도의 함수로 분리해봅시다.

```c
#include <stdlib.h>

// [!code ++:3]
void print_task(int id, char name[], char status[]) {
  printf("%d: %s\t%s\n", id, name, status);
}
```

이것은 함수를 정의<L>define</L>한다고 부릅니다.
여기서 `void`는 돌려줄 값이 없다는 것을 의미합니다.
그리고 괄호 안은 함수의 파라미터<L>parameter</L>라고 부르며, 함수가 호출될 때 함께하는 변수입니다.
여기서는 정수 `id`와 스트링 `name`, `status`를 받습니다.

그리고 아래의 `main` 함수에서는 이렇게 만든 함수를 사용합니다.

```c
  // [!code --:1]
  printf("%d: %s\t%s\n", id, name, status);
  // [!code ++:1]
  print_task(id, name, status);
```

이렇게 괄호 안에 든 값을 호출 인자<L>argument</L>라고 부르며, 이런 표현을 함수 호출이라고 부릅니다.
실행하면 이전과 같을 것입니다. (한번 해보세요.)

## 아래에서 함수 구현하기: 프로토타입

컴파일러는 소스 코드를 위에서 아래로 읽기 때문에, `print_task` 함수를 먼저 읽고 `main` 함수를 읽습니다.
따라서 `main` 함수가 `print_task` 함수를 호출할 수 있게 됩니다.
반대로 말하면, `print_task` 함수는 `main` 함수 아래에서 정의할 수는 없다는 뜻입니다.

사람 또한 소스 코드를 위에서 아래로 읽기 때문에, 실제 실행 순서대로 코드가 배치되는 것이 이해에 도움을 줍니다.
그렇다면 `print_task` 함수는 어떻게 `main` 아래에서 정의할 수 있을까요?
만약 그렇게 하고 싶다면, `print_task`가 적어도 어떤 모습인지 컴파일러에게 알려줘야 합니다.
이는 다음과 같이 할 수 있습니다.

```c
// [!code --:3]
void print_task(int id, char name[], char status[]) {
  printf("%d: %s\t%s\n", id, name, status);
}
// [!code ++:1]
void print_task(int id, char name[], char status[]);
```

이런 선언을 함수의 프로토타입<L>prototype</L>이라고 부릅니다.
이를 통해 다음과 같이 함수의 구현을 `main` 함수 아래로 내릴 수 있습니다.

```c
int main(void) {
  int id = 1;
  char name[] = "Learn C";
  char status[] = "To-do";

  print_task(id, name, status);

  return EXIT_SUCCESS;
}

// [!code ++:3]
void print_task(int id, char name[], char status[]) {
  printf("%d: %s\t%s\n", id, name, status);
}
```

실행해보면 이전과 같을 것입니다.

## 연관있는 변수 모으기: 구조체

생각해보면 `id`, `name`, `status` 같은 변수는 하나의 할 일을 구성합니다.
따라서 따로 떨어진 변수를 만드는 대신, 새로운 타입을 만들어 이것들을 할 일이라는 의미로 모을 수 있습니다.

```c
#include <stdlib.h>

// [!code ++:5]
struct task {
  int id;
  char* name;
  char* status;
};
```

이것은 `task`라는 구조체<L>structure</L>를 만들었다고 부르며, `id`, `name`, `status` 같은 것은 이 구조체의 필드<L>field</L>라고 부릅니다.
여기서 `char *`라는 새로운 타입이 등장했는데, 추후 살펴보겠지만 지금은 스트링을 나타내는 타입이라고 이해하고 넘어가겠습니다.

이렇게 만든 구조체는 다음과 같이 사용합니다.
먼저, `main` 함수에서 구조체를 초기화해봅시다.

```c
  struct task first = {
    .id = 1,
    .name = "Learn C",
    .status = "To-do",
  };
```

구조체는 새로운 타입이므로 `struct task`가 타입이 됩니다.
그리고 `first` 변수는 위와 같은 코드로 각 필드를 초기화합니다.

이 구조체 변수는 다음과 같이 점을 이용해 필드에 접근할 수 있습니다.

```c
  printf("%d: %s\t%s\n", first.id, first.name, first.status);
```

이를 이용해 전체적인 코드를 수정해봅시다.
먼저 함수의 프로토타입은 구조체를 받도록 고칩니다.

```c
// [!code --:1]
void print_task(int id, char name[], char status[]);
// [!code ++:1]
void print_task(struct task t);
```

이에 따라 `main` 함수에서 함수의 호출 또한 수정합니다.

```c
  // [!code --:1]
  print_task(id, name, status);
  // [!code ++:1]
  print_task(first);
```

마지막으로 함수 구현은 앞서 소개한 내용을 이용해 직접 수정해보시기 바랍니다.

## 타입에 새로운 이름 짓기: `typedef`

앞서 구조체 타입을 새로 만들긴 했지만, 매번 `struct` 키워드를 붙이는 것은 약간 귀찮습니다.
C 언어에서는 기존 타입에 새 이름을 지을 수 있습니다.

```c
// [!code ++:1]
typedef struct task task;
struct task {
  int id;
  char* name;
  char* status;
};
```

이것은 우리의 긴 이름을 `task`로 줄이겠다는 뜻입니다.
또는 다음과 같이 줄여서 쓸 수도 있습니다.

```c
// [!code --:6]
typedef struct task task;
struct task {
  int id;
  char* name;
  char* status;
}
// [!code ++:5]
typedef struct task {
  int id;
  char* name;
  char* status;
} task;
```

이것은 구조체 타입을 선언함과 동시에 이름을 지어줍니다.
여기서는 이 방법을 쓰겠습니다.

그러면 `main` 함수에서도 이 새로운 이름을 사용할 수 있습니다.

```c
  // [!code --:1]
  struct task first = {
  // [!code ++:1]
  task first = {
    .id = 1,
    .name = "Learn C",
    .status = "To-do",
  };
```

함수의 프로토타입 또한 이와 같습니다.

```c
// [!code --:1]
void print_task(struct task t);
// [!code ++:1]
void print_task(task t);
```

함수 정의는 직접 수정해보시기 바랍니다.

## 같은 타입 모으기: 배열

구조체가 타입이 다른 것들의 의미있는 모임이라면, 배열은 타입이 같은 것들을 모은 것입니다.
여기서는 하나의 할 일이 아닌 여러 개인 경우로 일반화하고자 합니다.

먼저, 두 번째 할 일을 추가해봅시다.
앞서 소개하기도 했지만, 배열은 다음과 같이 만들 수 있습니다.

```c
  // [!code --:5]
  task first = {
    .id = 1,
    .name = "Learn C",
    .status = "To-do",
  };
  // [!code ++:15]
  task tasks[] = {
    {
      .id = 1,
      .name = "Learn C",
      .status = "To-do",
    }, {
      .id = 2,
      .name = "Get a coffee",
      .status = "To-do",
    }, {
      .id = 3,
      .name = "Take a walk",
      .status = "To-do",
    }
  };
```

이렇게 할 일 여러개를 하나의 배열로 만들었습니다.
배열 안에 있는 각각의 값은 원소<L>element</L>라고 부르며, 다음과 같이 접근할 수 있습니다.

```c
  // [!code --:1]
  print_task(first);
  // [!code ++:1]
  print_task(tasks[0]);
```

이 코드에서 `[0]`은 배열의 첫 번째 원소를 가져오는 것입니다.
이러한 순번을 배열의 인덱스<L>index</L>라고 부릅니다.
즉 C 언어에서 인덱스는 `0`부터 시작합니다.
사실 대부분의 프로그래밍 언어가 이 관습을 따릅니다.
이것을 첫 번째 원소로부터 떨어진 거리로 이해하는 것을 권합니다. (실제로도 그러한데, 그 이유는 추후 살펴봅니다.)

한번 인덱스를 바꿔서 결과를 확인해보면 어떨까요?

```c
  // [!code --:1]
  print_task(tasks[0]);
  // [!code ++:1]
  print_task(tasks[1]);
```

실행 결과는 다음과 같습니다.

```shellsession
2: Get a coffee Done
```

배열은 두 개의 원소이므로, 이것보다 큰 인덱스는 의미상 말이 되지 않습니다.
하지만 C 언어는 그것을 막지 않습니다.
C 언어의 기본 철학은, 프로그래머는 자신이 무엇을 하는지 정확히 알고 있다는 것입니다.

여기서 시험삼아 한번 시도해보세요.
그리고 왜 그런 값이 나오는지 생각해보세요.
대신 절대 다른 곳에서는 그런 코드를 쓰면 안 됩니다.

## for 반복문

우리의 프로그램은 모든 할 일을의 목록을 보여주는 것입니다.
따라서 하나씩 출력하는 대신, 모든 원소를 출력해야 합니다.

가장 간단한 방법은 다음과 같이 일일이 출력하는 것입니다.

```c
  // [!code ++:1]
  print_task(tasks[0]);
  print_task(tasks[1]);
  // [!code ++:1]
  print_task(tasks[2]);
```

그러면 결과는 다음과 같습니다.

```shellsession
1: Learn C      To-do
2: Get a coffee Done
3: Take a walk  To-do
```

그런데 만약 이것보다 더 늘어나면 어떨까요?
더 나아가서, 개수를 미리 알 수 없다면 어떨까요?
지금은 가짜로 할 일을 구현해놨기 때문에 개수를 미리 정해놨지만, 나중에는 다른 곳에서 읽어 가져올 것이기 때문에 미리 알 수 없습니다.

개수가 세 개라고 알아냈다면, 이를 이용해 다음과 같이 반복문을 사용할 수 있습니다.

```c
  // [!code --:3]
  print_task(tasks[0]);
  print_task(tasks[1]);
  print_task(tasks[2]);
  // [!code ++:3]
  for (size_t i = 0; i < 3; i++) {
    print_task(tasks[i]);
  }
```

여기서 `size_t` 타입은 크기를 나타내는 타입입니다. 추후 나중에 살펴볼 것입니다.

## 리팩토링: 배열을 파라미터로

생각해보면 여러 개의 할 일을 프린트 하는 일 또한 하나의 작은 일입니다.
이를 함수로 분리해봅시다.
먼저 프로토타입을 추가합니다.

```c
// [!code ++:1]
void print_tasks(task tasks[], size_t size);
void print_task(task t);
```

그리고 `main` 함수 아래에 구현을 추가합시다.

```c
int main(void) {
  // ...

  // [!code --:3]
  for (size_t i = 0; i < size; i++) {
    print_task(tasks[i]);
  }
  // [!code ++:1]
  print_tasks(tasks, 3);
}

// [!code ++:5]
void print_tasks(task tasks[], size_t size) {
  for (size_t i = 0; i < size; i++) {
    print_task(tasks[i]);
  }
}
```
